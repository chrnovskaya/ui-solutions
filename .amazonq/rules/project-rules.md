# Правила проекту

1. **Ніякого хардкодингу** — використовувати змінні, константи, CSS custom properties замість захардкоджених значень.
2. **Не вгадувати** — якщо не впевнений, не знаєш як щось робити або не розумієш задачу — перепитати у користувача, а не робити наугад.
3. **Доповнювати правила** — якщо користувач нагадує про щось що має бути правилом, одразу дописувати це в правила.
4. **Архітектура фічі** — кожна фіча в `src/features/<feature-name>/` має таку структуру:
   ```
   feature-name/
   ├── hooks/          — React-хуки
   ├── lib/            — утиліти, хелпери, чисті функції
   ├── types/          — інтерфейси та типи (index.ts)
   ├── ui/             — підкомпоненти
   ├── views/          — сторінки/в'юшки (композиція компонентів для роутів)
   ├── FeatureName.tsx — головний компонент (композиція)
   └── index.ts        — публічний API фічі (реекспорт)
   ```
